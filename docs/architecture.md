# アーキテクチャ方針

このドキュメントは、nonono プロジェクトの目標と、それに基づくアーキテクチャ上の決定事項をまとめたものである。

## プロジェクトの目標

- ノノグラムの問題を提供し続けること(コンテンツの継続供給)
- PCでもスマホでも快適な操作UIを提供すること
- 運用費は無料にすること

## 決定事項

### ユーザーデータ

ユーザーアカウント機能は持たない。進捗データのみ `localStorage` に保存する。

### 問題データの表現

問題(盤面)データは、lz-string などで可逆圧縮した文字列としてURLに埋め込んで表現できるようにする。
問題の共有などが単一のURLで行えるようにする想定。

### ユーザー作問と新着一覧

運営による出題ではなく、ユーザーが作問してURLで共有する形をメインとする。ただし「新着問題」をサイトに一覧表示するため、新着100件程度を記録する軽量なバックエンド(KV等)は導入する。

### 画面構成とルーティング

- `/` — トップ、新着問題のインデックス
- `/puzzles/[data-string]` — 問題回答。`data-string`はlz-stringの`compressToEncodedURIComponent`でエンコードした盤面データ
- `/puzzles/new` — 問題作成
- `/mypage` — 直近にプレイした問題の進捗1件と、自分で投稿した問題一覧（どちらもlocalStorageに保持）

### ホスティング

**Vercel** を採用する。Next.js公式プラットフォームであり追加設定なしにApp Router全機能が動く。新着一覧用のKVもVercel Marketplace経由のUpstash Redis連携の無料枠で賄う。

Vercel Hobbyプランでは商用利用が禁止な点に注意。広告掲載なども行えないが、広告はユーザ体験の阻害になる上に効果も薄いであろうことを考えると、そもそも広告を掲載すべきでない。

- IPベースのレート制限
- 盤面バリデーション(既存ソルバーで解ける正しい盤面かのチェック)

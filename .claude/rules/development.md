# 原則

- 質問に対して作業で返してはならない

# 開発プロセス

## ToDoによる計画

- 実装前に必ず TaskCreate/TaskUpdate ツールでタスク計画を作成
- `./agent-docs` 配下に作成したタスク計画を出力、必要に応じて更新する

## TDD

開発は原則、テスト駆動開発（TDD）の手法に従って、以下のサイクルで開発を進める:

1. **Red**: 失敗するテストを先に書き、`npm run test:run` で失敗を確認する
2. **Green**: テストが通る最小限の実装を書く
3. **Refactor**: `/simplify` でレビューし、必要に応じて対応する

## コミット

コミットメッセージは、そのコミットに載る変更を `git diff` で確認し、それに即した内容を記述すること。

# コーディング規約

- コードスタイルは next lint で強制。hook により Edit/Write 時に自動実行されるのでエラーが出たら適切に修正すること
- 変数名は一見して実態が想像できる名前をつける。`u`, `i` などの省略形は原則避けること
